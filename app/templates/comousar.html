Metodologia de Desenvolvimento

Sobre esse trabalho mostaremos como foi desenvolvido, mostrando o modelo de evasão utilizado, o processo de levantamento de dados, levantamento de requisitos, o modelo dimensional, a criação da rede bayesiana e a implementação do front end e back end .

1 Modelo de Evasão

Para esse trabalho foi considerado evadido todo aluno que tenha deixado o curso de TADS, ou seja, mesmo que o aluno permaneça na instituição em outro curso, ele será considerado evadido. Portanto, será considerado o conceito de evasão de curso.

2 Levantamento de dados

Os dados utilizados no sistema foram fornecido pelo Sistema Acadêmico do IFNMG-Januária (Cajuí). Onde foram contemplados as seguintes dados: Nome das disciplinas, em qual período do curso foi dado aquela disciplinas e quais as situações finais da disciplina, para cada aluno que já se matriculou no curso de Tecnologia em Análise e Desenvolvimento de Sistemas do IFNMG – Campus Januária até no primeiro semestre o ano de 2018.

3 Levantamento de Requisitos

Essa etapa configurou se em analisar quais dados referentes às disciplinas que seriam necessários extrair da base de dados do Cajuí para que os menos pudessem serem utilizados na etapa de construção da rede bayesiana.

4 Modelagem Dimensional

A etapa seguinte foi a de construção do Data Warehouse dos dados adquiridos do Cajuí, com esses dados foi criado um Dataframe. Foi recebido o banco de dados bruto que tem o PostgreSQL como seu sistema gerenciador de banco de dados, a partir desse dados foram feita a mineração de separado os dados referentes às disciplinas, tais dados já citados acima. Com os dados extraídos do banco de dados original criou se um Dataframe no formato de texto Comma-separated values( cvs).

5 Metodologia CRISP-DM

Como metodologia de estudo aplicamos no decorrer deste trabalho as etapas previstas no CRISP-DM, já abordadas no capítulo 3. CRISP-DM é uma metodologia de mineração de dados em formato cíclico, composto por 6 fases, que direcionam a descoberta do conhecimento para tomada de decisão sobre dados em grande volume. CRISP-DM foi concebido no final de 1996 por três “veteranos” do mercado de mineração de dados. A metodologia CRISP-DM é descrita em termos de um modelo de processo hierárquico, consistindo de conjuntos de tarefas.(CHAPMAN, 2000).

6 Criação da Rede Bayesianas

Foi utilizado o Classificador Naive Bayes, com os dados obtidos na etapa de mineração foi criado um conjunto de dados de treinamento de possíveis situações das disciplinas, foi feita a classificação desses dados em todas as possibilidades. O passo seguinte foi gerar a tabela de frequência relativa para cada situação em cada disciplina. Em seguido foi feito a tabela de probabilidades das seguintes situações: Aprovado, Reprovado, Matriculado e Desistente. Para calcular a probabilidade foi utilizado o Classificador Naive Bayes, onde o resultado do cálculo será o resultado da classificação. Abaixo podemos identificar a função para calcular a probabilidade de um aluno obter uma situação, respondendo a seguinte pergunta: Qual a probabilidade do aluno ser aprovado dado que ele foi reprovado na disciplina de algoritmos?
P(Período = 3, SituacaoDisciplina = Aprovado | Disciplina = "Algoritmos")
De acordo com a inserção dos dados e a realização dos cálculos a classificador ficar mais rápido é com uma maior nível de confiança uma vez que ele aprende com cada classificação.

7 Implementação do Front End e Back End

Na criação do Front End foi utilizado a linguagem python e html. Para a estilização foi utilizado o bootstrap para os menus e para que o sistemas possa ser responsivo. Já o Back End foi utilizado o Python e Flask, o trabalho foi utilizado a arquitetura MVC(Model–view–controller) um padrão de arquitetura para organizar a aplicação, dividindo a mesma em 3 camadas, assim facilita a construção e a manutenção da aplicação e por fim foi utilizado a linguagem de programação Java Script para fazer algumas validações de formulários. .
